## 1. Introduction & Design Philosophy

The Errum V2 e-commerce frontend is a high-performance, premium storefront designed to provide a "boutique" shopping experience. Built on **Next.js 15 (App Router)** and **React 19**, it utilizes a **"Premium Dark" (Ink & Gold)** aesthetic. The system is engineered for multi-store scalability, handling complex product variations through a sophisticated "Mother-Child" grouping logic.

### 1.1 Visual Identity & Typography
The storefront's "Luxury Dark" theme is achieved through a multi-layered visual stack:
- **Ink-Black Theme:** The root background is hardcoded to `#0d0d0d`.
- **Atmospheric Depth:** Fixed radial gradients (Glow Blobs) in Gold and Blue create a sense of depth and luxury.
- **Grid Texture:** A persistent `56px x 56px` grid pattern provides a high-tech, modern structural feel.
- **Typography Stack:** 
    - **Primary Headings:** *Cormorant Garamond* (High-end Serif) for a traditional luxury feel.
    - **Body & Controls:** *Jost* (Modern Sans-serif) for high readability.
    - **Metadata & Technicals:** *DM Mono* for SKU labels, order numbers, and timestamps.

## 2. Core Infrastructure & State Management

### 2.1 Global Layout (`app/e-commerce/layout.tsx`)

### 2.2 Cart Context & Logic (`app/e-commerce/CartContext.tsx`)
The cart system is more than a simple array; it’s an intelligent sync layer.
- **Image Picking Algorithm:** Implements `pickImage` logic that scans product variants to find the best primary image, thumbnails, or fallback paths. It prioritizes `is_primary` flags but falls back to the first available media item.
- **Event-Driven Sync:** Dispatches custom `cart-updated` events to ensure that if a user opens the store in multiple tabs, the bag stays synchronized.
- **Backend Validation:** Provides a `validateCart` bridge to the backend, which performs real-time stock checks before allowing checkout.

## 4. The Mother-Child Product Architecture

A central innovation in Errum V2 is the **Mother-Child Pattern** implemented in `lib/ecommerceProductGrouping.ts`.

### 4.1 The Grouping Algorithm
The storefront avoids displaying repetitive cards for the same product in different sizes. Instead:
1.  **Normalization:** Product names are stripped of size/color suffixes to identify the "Base Identity."
2.  **Representative Selection:** The system picks one variant (usually the one with the highest stock) to be the "Mother" card.
3.  **Variant Counter:** A dynamic badge (e.g., "+12 sizes available") is generated by counting siblings in the same group.

### 4.2 Media Propagation
One of the most frequent e-commerce data issues is missing photos for specific size variants. Errum V2 solves this by:
- **Image Crawling:** The grouping logic crawls all children in a group.
- **Fallback Assignment:** If a specific SKU lacks an image, it automatically "borrows" the primary image from the Mother product, ensuring no card ever shows a broken placeholder.

## 6. Checkout & Payment Security

### 6.1 Cart Integrity (`app/e-commerce/cart/page.tsx`)
- **Synchronous Write:** Before navigating to checkout, the app performs a synchronous write to `localStorage` to ensure the `selected-items` state is captured before the browser unloads the page.
- **Multi-Select Logic:** Users can selectively checkout items from their bag while leaving others for later.

### 6.2 SSLCommerz Payment Bridge (`components/ecommerce/SSLCommerzPayment.tsx`)
- **Transition UI:** A high-security overlay with a "Secure Encryption" animation appears during the gateway redirect.
- **Intent Recovery:** Stores `sslc_payment_intent` in `localStorage`. If a user's phone dies during payment, they can return to the site and the system will instantly check the transaction status.

## 12. Security Protocols & Token Management

### 12.1 Multi-Auth Security
The system maintains a strict separation between Admin and Customer scopes:
- **Interceptor Scoping:** The centralized `axiosInstance` checks the target URL. If it starts with `/customer` or `/checkout`, it injects the `auth_token`. Otherwise, it defaults to the Admin `authToken`.
- **Token Expiration Handling:** If an API call returns a `401 Unauthorized`, the system automatically attempts to resolve the session by falling back to "Guest Mode" or redirecting to the login portal.

### 12.2 Data Sanitization
- **Slugification:** All user-provided search terms and category names are routed through a `slugify` utility to prevent XSS via URL parameters.
- **Sanitized HTML:** Receipt previews and product descriptions are rendered using standard React escaping or trusted `srcDoc` iframes for the thermal printer previews.

## 13. Performance & Code Splitting

- **Suspense Boundaries:** Critical for pages using `searchParams` (Checkout, Search) to avoid SSR hydration mismatches.
- **Image Proxies:** All media is routed through `/api/proxy-image` to resolve CORS issues from external storage disks.
- **Tree Shaking:** All Lucide icons are imported as named members to minimize the final JS bundle size.
